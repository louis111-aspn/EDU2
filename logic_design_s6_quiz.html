Q1 Where
시나리오
전국 500개 매장의 일별 판매 데이터가 DB에 누적되어 있다 (총 5,000만 행). 경영진이 "지난 3년간 지역별·월별 매출 추이"를 볼 수 있는 리포트를 요청했다. 개발팀에서 아래와 같은 구현 방안을 제안했다. 
-- 제안 A: 앱 서버에서 처리
all_rows = db.execute("SELECT * FROM sales")
result = aggregate_in_python(all_rows)

-- 제안 B: DB에서 집계 완결
SELECT region, DATE_TRUNC('month', sale_dt),
       SUM(amount)
FROM sales
GROUP BY 1, 2
ORDER BY 1, 2;
어떤 제안이 맞는가, 그리고 그 핵심 이유는?
Q2 How
시나리오
물류 시스템에서 화물의 운송비를 산정한다. 운송비는 무게 구간(0–10kg, 10–30kg, 30kg 초과)과 배송 유형(일반, 특급, 당일)의 조합으로 결정되며, 총 9가지 경우가 존재한다. 아래 두 가지 구현 방식이 제시되었다. 
-- 방식 X: 중첩 IF
if weight <= 10:
    if   type == '일반': fee = 3000
    elif type == '특급': fee = 5000
    else:              fee = 8000
elif weight <= 30:
    if   type == '일반': fee = 5000
    elif type == '특급': fee = 8000
    else:              fee = 12000
else: ...

-- 방식 Y: Lookup Table
FEE_TABLE = {
  ('S', '일반'): 3000, ('S', '특급'): 5000, ('S', '당일'): 8000,
  ('M', '일반'): 5000, ('M', '특급'): 8000, ('M', '당일'): 12000,
  ('L', '일반'): 8000, ('L', '특급'): 13000, ('L', '당일'): 20000,
}
weight_band = 'S' if weight<=10 else 'M' if weight<=30 else 'L'
fee = FEE_TABLE.get((weight_band, type), 0)
6개월 뒤 "프리미엄" 배송 유형이 추가된다면, 어느 방식이 유지보수가 쉬운가?

Q5 Pattern
시나리오
매일 새벽 2시에 전날 주문 데이터를 집계해 daily_stats 테이블에 저장하는 배치가 있다. 어느 날 서버 장애로 배치가 중간에 중단됐고, 복구 후 다시 실행했다. 아래 두 가지 구현 중 재실행이 안전한 방식은? 
-- 구현 P
INSERT INTO daily_stats (dt, order_cnt, revenue)
SELECT order_dt, COUNT(*), SUM(amount)
FROM orders
WHERE order_dt = '2024-03-15'
GROUP BY order_dt;

-- 구현 Q
DELETE FROM daily_stats
WHERE dt = '2024-03-15';

INSERT INTO daily_stats (dt, order_cnt, revenue)
SELECT order_dt, COUNT(*), SUM(amount)
FROM orders
WHERE order_dt = '2024-03-15'
GROUP BY order_dt;
