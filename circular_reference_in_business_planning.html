순환참조란 무엇인가
경영계획에서 순환참조가 발생하는 구조적 원인과 개념을 이해합니다
상호의존 변수(Mutual Dependency)란

경영계획에서 순환참조(Circular Reference)는 단순한 수식 오류가 아닙니다. 계획 변수 A가 변수 B를 결정하고, 변수 B가 다시 변수 A에 영향을 미치는 상호의존 구조가 계획 모델 안에 내재되어 있기 때문에 발생합니다.

핵심 개념
결과변수(Output)가 동시에 투입변수(Input)인 구조 — 이것이 경영계획 순환참조의 본질입니다.

  • 존재하지 않는 셀을 참조

  • Excel이 경고를 주지만 허용 가능
  • 결과가 나오기 때문에 문제를 인지하기 어려움
주의

케이스 01 — 유동성·자금 (이자비용 순환)

가장 전형적인 순환참조 케이스. PL 계획 → CF 부족 → 차입 → 이자비용 → PL 변경의 경로로 순환이 형성됩니다.

PL 계획 (영업이익 확정)
현금흐름 계획 수립 (영업CF + 투자CF)
자금 부족/잉여 판단 → 차입 규모 결정
이자비용 변동 (차입금 × 금리)
당기순이익 변경
♻️ 다시 PL 계획에 영향 → 순환
실무 맥락
연간 경영계획 수립 시 재무팀이 이자비용을 전년 실적값으로 먼저 입력 후, 자금계획 수립 과정에서 차입 규모가 변동되면 이자비용이 달라지고, 이것이 순이익·배당·CF에 연쇄 영향을 줍니다.
미처리 시 리스크
계획 대비 실제 이자비용 편차 발생 → 순이익 목표 미달 → 성과급 KPI 연동 문제로 확대될 수 있습니다.
● 발생빈도 매우높음
케이스 02 — 인건비·성과급

성과급이 영업이익의 일정 %로 연동되는 구조에서, 성과급 자체가 영업이익을 구성하는 비용이 되어 순환이 발생합니다.

매출 계획 확정
영업이익 계획 (성과급 제외 기준)
성과급 지급률 결정 (영업이익 연동 KPI)
인건비(성과급) 증가 → 영업이익 감소
성과급 지급률 재산정
♻️ 다시 영업이익에 영향 → 순환
실무 맥락
단순 1단계 KPI는 해석적으로 풀 수 있습니다. 그러나 영업이익 → 성과급 → 영업이익 → 세후이익 연동 장기인센티브(LTI)처럼 다단계 KPI가 겹치면 순환의 깊이와 경로 수가 급증합니다.
미처리 시 리스크
HR이 계획한 성과급과 재무가 계획한 영업이익이 상호 모순 — 연간 계획 확정 후 보정 과정에서 어느 팀의 수치가 기준인지 충돌이 발생합니다.
● 발생빈도 중
케이스 03 — 법인세

단순 세율 적용이면 순환이 없습니다. 그러나 최저한세, 이월결손금 공제, 연결납세 구조에서는 세금 자체가 과세표준에 역영향을 줍니다.

세전이익 계획
법인세 산출 (세전이익 × 유효세율)
세무조정 항목이 세전이익에 역영향
(최저한세 / 이월결손금 / 이연법인세)
당기순이익 변경 → BS 이익잉여금 변동
♻️ 과세표준 재산정 → 순환
실무 맥락
IFRS 기준 이연법인세 산출 시 일시적 차이(Temporary Difference)가 세후이익을 바꾸고, 이것이 다시 이연법인세 자산·부채에 영향을 주는 미세 순환이 존재합니다. 세무팀과 재무팀이 분리된 조직에서 더욱 두드러집니다.
미처리 시 리스크
계획 유효세율이 실제와 괴리 → 순이익 목표 편차 → 배당 계획 연동 오류로 이어집니다.
● 발생빈도 중
케이스 04 — CAPEX·감가상각

제조업·장치산업에서 두드러집니다. CAPEX가 크고 감가상각 기간이 길수록 투자 결정과 수익성 계획이 상호 제약합니다.

매출 성장 계획
생산능력 확대를 위한 CAPEX 계획
신규 자산 취득 → 감가상각비 증가 → 영업이익 감소
잉여현금(FCF) 감소 → CAPEX 조달 가능 규모 재산정
CAPEX 축소 → 매출 성장 재검토
♻️ 다시 매출 계획에 영향 → 순환
실무 맥락
3~5개년 장기계획에서 더욱 복잡하게 나타납니다. 연도별 CAPEX가 전년도 감가상각비와 연동되고, 이것이 다시 다음 연도 영업이익 계획에 영향을 주는 다년도 순환 구조가 형성됩니다.
미처리 시 리스크
CAPEX를 과대 계획 후 FCF 부족으로 조달 실패 → 투자계획 전면 재수립 상황 발생. 또는 보수적 CAPEX로 성장 목표 달성 불가.
● 발생빈도 높음
케이스 05 — 운전자본(Working Capital)

매출 계획이 채권·재고 증가를 유발하고, 이것이 CF에 영향을 주어 차입을 늘리고, 이자비용이 다시 순이익에 영향을 주는 구조입니다.

매출 계획
매출채권 증가 (DSO 가정 적용)
영업CF 감소 → 단기차입 증가
이자비용 증가 → 순이익 감소
매출 목표 재검토 (수익성 기준 미달)
♻️ 다시 매출 계획에 영향 → 순환
실무 맥락
재고 측면 변형도 존재합니다: 매출 계획 → 생산계획(재고 목표) → 원재료 매입 증가 → 매입채무 증가 → 운전자본 부담 → CF 영향 → 매출 계획 조정. 유통·제조업에서 빈번합니다.
미처리 시 리스크
공격적 매출 목표 달성 시 오히려 운전자본 급증으로 현금 위기 발생. "매출은 늘었는데 현금이 없는" 상황의 계획 단계 재현입니다.
● 발생빈도 중
케이스 06 — 배당·자본정책

배당성향이 정책적으로 고정된 상장사·지주회사에서 발생합니다. 자사주 매입이 추가되면 순환 경로가 더욱 복잡해집니다.

당기순이익 확정
배당 계획 (배당성향 % 적용)
배당금 지급 → 현금 유출 → 차입 증가
이자비용 증가 → 당기순이익 감소
배당 재산정 (배당성향 동일 적용 시)
♻️ 배당 감소 → 순이익 재변동 → 순환
실무 맥락
배당성향 30% 정책을 유지하는 상장사에서, 배당 지급을 위한 차입이 이자비용을 높이고, 이것이 배당 재원인 순이익을 낮추어 결국 배당이 줄어드는 자기모순적 계획이 형성됩니다.
미처리 시 리스크
IR 공시 기준 배당 예고액과 실제 지급 가능 배당액 간의 괴리 → 주주 기대관리 실패 리스크.
● 사업 구조 의존
케이스 07 — 환율·외화

글로벌 사업 비중이 높은 기업에서 발생합니다. 환율 시나리오가 복수일 때 순환의 경로 수가 기하급수적으로 증가합니다.

매출 계획 (외화 포함)
환율 가정 적용 → 원화 환산 매출 확정
외화 차입 규모 결정 (Natural Hedge 목적)
환율 변동 시 외환손익 발생 → 순이익 변동
헤지 규모 재산정 → 외화 포지션 변경
♻️ 환산 매출·비용 재계산 → 순환
실무 맥락
수출 비중이 높은 제조사에서 매출의 70%가 USD 기준일 때, 환율 가정 1% 변동이 매출 → 헤지 → 금융비용 → 순이익 경로로 연쇄 영향을 미칩니다. Base/Upside/Downside 3개 시나리오가 있으면 순환 경로가 3배로 늘어납니다.
미처리 시 리스크
환율 시나리오별 순이익 편차가 비선형적으로 증폭 → 경영진에게 제시하는 민감도 분석 결과의 신뢰성 저하.
Excel 방식의 작동 원리와 한계
"반복계산 허용"이 왜 충분하지 않은지 단계별로 확인합니다
Excel 반복계산 메커니즘

Excel은 순환참조 발생 시 경고를 표시하고, 옵션 설정에서 반복 계산 허용을 활성화하면 숫자가 나옵니다.

설정 경로:
파일 → 옵션 → 수식
→ ✅ 반복 계산 사용
→ 최대 반복 횟수: 100 (기본값)
→ 최대 변화값: 0.001 (기본값)
문제의 시작
이 설정이 활성화되면 Excel은 수렴했는지 여부와 관계없이 최대 반복 횟수에 도달하면 계산을 멈추고 그 시점의 값을 표시합니다.
구조적 한계 5가지 — 항목을 눌러 확인하세요
① 수렴 여부가 비가시적 — 결과 신뢰성 미보증 ▼
Excel의 반복계산은 수렴했는지, 발산했는지를 셀에 표시하지 않습니다. 담당자는 결과값을 보고 "맞겠지"라고 판단합니다.

경영진에게 보고되는 순이익이 3번째 반복값인지 100번째 반복값인지 추적이 불가능합니다. 두 값이 같다는 보장도 없습니다.
② 초기값 의존성 — 비결정론적 모델 ▼
반복계산의 수렴 결과는 초기값(이자비용 시작값)에 따라 달라질 수 있습니다.

전년도 이자비용을 초기값으로 쓰느냐, 0을 쓰느냐에 따라 최종 수렴값이 미세하게 달라집니다. 이를 인지하는 실무자는 거의 없으며, 매년 계획의 초기 조건이 다르면 비교 가능성이 훼손됩니다.
③ 시나리오 전환 시 오염 — 재현 불가 ▼
Base → Downside 시나리오로 전환 시, 순환참조 셀이 이전 시나리오의 반복계산 잔재값을 초기값으로 물고 들어갑니다.

파일을 열고 닫는 순서, 계산 트리거 순서에 따라 결과가 달라지는 비결정론적 모델이 됩니다. 동일 파일을 두 사람이 각자 열었을 때 결과가 다를 수 있습니다.
④ 감사 추적 불가 — 내부통제 리스크 ▼
순환참조가 허용된 Excel 모델은 수식 추적(Formula Auditing)이 불완전합니다.

"이 숫자가 왜 이렇게 나왔는가"를 재현 가능한 방식으로 설명하기 어렵습니다. 상장사나 공시 연계 계획 수립 조직, 내부감사 대상 계획에서는 직접적인 리스크입니다.
⑤ 협업 구조에서 즉시 붕괴 — 확장성 없음 ▼
한 명이 파일을 관리할 때는 작동합니다. 그러나 PL은 기획팀, CF는 재무팀, 조달계획은 자금팀이 각자 시트를 갖는 순간, 순환참조는 파일 분리와 함께 끊어집니다.

통합 파일 구조를 강제하거나, 링크 참조 오류를 상시 관리해야 하는 운영 부담이 발생합니다. 담당자가 교체되면 이 구조를 이해하는 사람이 없어지는 암묵지 의존 리스크도 있습니다.
핵심 판단
Excel 순환참조 허용은 단일 담당자의 단일 파일 환경에서만 통제 가능한 방식입니다. 조직 규모의 경영계획 수립에는 재현성·감사가능성·협업 안정성 측면에서 구조적 한계가 있습니다.
SAC Planning의 해결 접근
SAC Planning이 순환참조를 다루는 구조적 방식을 개념 수준에서 이해합니다
SAC가 Excel과 근본적으로 다른 점

SAC Planning은 순환참조를 "허용"하는 것이 아니라, 계산 순서를 설계자가 명시적으로 제어함으로써 순환 자체가 발생하지 않도록 구조화합니다.

Excel 반복계산 허용 방식
❌수렴 여부를 사용자가 확인 불가
❌계산 순서가 Excel 내부 엔진에 의해 결정
❌초기값·파일 오픈 순서에 따라 결과 변동
❌시나리오 전환 시 이전 값 오염
❌파일 분리 시 순환 경로 단절
SAC Planning 구조적 제어 방식
✅계산 단계별 실행 순서를 설계자가 명시
✅각 단계의 입력·출력이 명확히 분리
✅수렴 반복 횟수와 임계값을 로직으로 제어
✅버전 분리로 시나리오 간 오염 차단
✅다수 사용자 동시 입력 환경에서도 정합성 유지
접근 1. Data Action Sequence — 계산 순서 명시적 제어

SAC의 Data Action Sequence는 여러 계산 로직(Script)을 지정된 순서대로 순차 실행하는 기능입니다. 순환참조 해소의 핵심 도구입니다.

Step 1. PL Script 실행 → 영업이익 산출
Step 2. CF Script 실행 → 영업CF / 투자CF 산출
Step 3. 조달계획 Script → 차입 규모 / 이자비용 산출
Step 4. PL 이자비용 라인 업데이트
Step 5. 수렴 판단 (전 회차 대비 순이익 델타 체크)
수렴 미달 시 Step 1~5 재실행 (최대 3회) — 로그 기록 포함
Excel과의 차이
반복 횟수, 수렴 기준, 각 단계의 실행 결과가 모두 시스템에 기록됩니다. "몇 번 만에 수렴했는가", "수렴하지 못했는가"를 감사 추적할 수 있습니다.
접근 2. 버전 분리 운영 — 시나리오 오염 차단

순환 경로의 각 계획을 별도 버전으로 관리한 후, 최종 수렴 시 통합 버전에 병합합니다.

버전용도
V_PL PL 계획 버전영업이익 기준 계획
V_CF CF 계획 버전현금흐름 계획
V_FIN 조달계획 버전차입/이자비용 계획
V_FINAL 통합 버전수렴 확정 후 병합
설계 원칙
각 버전은 독립적으로 유지되어 시나리오 전환 시 이전 버전의 값이 오염되지 않습니다. 통합 버전은 수렴 확정 후에만 업데이트하는 "잠금(Freeze)" 구조로 운영합니다.
접근 3. 수렴 가시화 — 감사 가능한 계획 수립

SAC에서 반복 수렴 로직을 설계할 때, 각 반복 회차의 핵심 변수값을 별도 차원에 기록하면 수렴 과정 자체를 추적할 수 있습니다.

수렴 가시화가 가능한 것들
회차별 순이익 변화량 / 수렴 도달 회차 / 수렴 실패 여부 및 CFO 확정 판단 이력 / 시나리오별 수렴 비교
전제 조건
SAC Planning에서의 이 접근은 Script Logic 설계 역량을 전제로 합니다. 개념 수준에서는 "계산 순서를 설계한다"는 원칙만 이해하면 충분하며, 구현 세부사항은 Senior 레벨에서 다룹니다.
